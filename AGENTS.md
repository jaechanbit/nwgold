# 프로젝트 작업 기준

## 문서 우선순위

작업 전 아래 문서를 확인한다.

1. `docs/CONTEXT.md`: 제품의 목적과 사용자
2. `docs/DESIGN.md`: 화면과 사용자 경험 기준
3. `docs/CONTENT.md`: 문구와 사진 작성 기준
4. `docs/ARCHITECTURE.md`: 기술 구조와 결정 사항
5. `docs/adr/`: 중요한 결정의 배경

문서와 구현이 다르면 임의로 구현을 바꾸지 않고, 의도를 확인한 뒤 문서와 구현을 함께 갱신한다.

## 작업 원칙

- 방문, 전화, 길찾기, 상담 등 매출과 가까운 행동을 우선한다.
- 카카오톡 채널 개설 전에는 전화를 주요 행동으로 사용한다.
- 카카오톡 채널 개설 후에는 카카오톡 문의를 주요 행동, 전화를 보조 행동으로 일관되게 표현한다.
- 점주가 상품과 영업 정보를 쉽게 수정할 수 있어야 한다.
- 모바일 화면을 먼저 설계한다.
- 귀금속의 가격, 소재, 중량 등 확인되지 않은 정보를 추측해서 표시하지 않는다.
- 경력, 기술, 후기 등 신뢰와 관련된 주장은 확인 가능한 근거와 함께 사용한다.
- 과장 광고나 근거 없는 최상급 표현을 사용하지 않는다.
- 개인정보는 필요한 만큼만 수집하며, 수집 목적을 사용자에게 알린다.
- 새 기능은 유지관리 비용과 점주가 실제로 사용할 수 있는지를 함께 검토한다.

## 품질 기준

- 주요 행동은 작은 모바일 화면에서도 쉽게 찾고 누를 수 있어야 한다.
- 주소, 영업시간, 연락처, 가격 표기에는 오류가 없어야 한다.
- 공개 상품에는 원화 가격과 가격 확인 시점을 함께 표시한다.
- 가격 또는 상품 상태가 확인되지 않은 콘텐츠는 게시하지 않는다.
- 가격 확인 후 14일이 지난 상품은 공개 화면에서 자동으로 숨긴다.
- 이미지에는 대체 텍스트를 제공하고, 텍스트와 배경의 대비를 확보한다.
- 공개 전 실제 기기에서 전화, 지도, 문의 링크를 확인한다.
- 변경 범위에 맞는 검사와 테스트를 실행한다.

## 아직 정하지 않은 사항

- 개발 기술과 배포 서비스
- 상품 정보 관리 방식
- 문의 접수 방식
- 고객 분석 도구와 개인정보 처리 범위
