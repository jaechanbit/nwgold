# 기술 구조

## 현재 상태

상품 사진과 내용을 매주 변경할 수 있는 관리형 정적 사이트로 구성한다. 기술 스택은 Astro, Sanity, Cloudflare Pages를 사용한다.

## 기술 스택

- 웹 프레임워크: Astro
- 콘텐츠 관리: Sanity Content Lake와 Sanity Studio
- 배포 및 호스팅: Cloudflare Pages
- 소스 저장소 및 배포 연결: GitHub
- 작성 언어: TypeScript
- 스타일: 프로젝트 내부 CSS와 디자인 토큰

방문자 페이지는 Astro로 정적 생성해 빠르게 제공한다. 관리자는 Sanity Studio에 로그인해 컴퓨터 웹 브라우저에서 상품과 영업 정보를 수정한다.

콘텐츠를 게시하면 Sanity 웹훅이 Cloudflare Pages의 배포 훅을 호출하고, 최신 콘텐츠로 사이트를 다시 빌드한다. 빌드에 실패하면 기존에 성공적으로 배포된 사이트를 유지한다.

## 선택 기준

- 점주가 상품, 사진, 영업시간을 수정하기 쉬운가
- 휴대전화 또는 익숙한 기기에서 코드 없이 수정할 수 있는가
- 모바일 성능과 검색 노출에 유리한가
- 배포와 유지관리 비용이 합리적인가
- 전화, 지도, 메시지 연결을 안정적으로 제공하는가
- 불필요한 개인정보를 저장하지 않을 수 있는가

## 필수 콘텐츠 관리 기능

- 로그인한 관리자만 콘텐츠 수정 가능
- 상품 추가, 수정, 숨김, 노출 순서 변경
- 상품명, 분류, 소개, 사진, 가격 표시 방식 관리
- 반지, 목걸이, 예물, 순금 등 상품 분류 관리
- 사진 업로드 시 웹에 적합한 크기와 형식으로 최적화
- 게시 전 미리보기 또는 초안 확인
- 잘못 수정했을 때 이전 내용 복구
- 영업시간과 공지 수정
- 한 명의 관리자가 컴퓨터 웹 브라우저에서 사용
- 공개 상품의 가격 및 가격 확인 시점 입력을 필수로 검증
- 가격 확인 후 14일이 지난 상품을 공개 결과에서 제외

첫 버전에서는 재고 수량, 주문, 결제 기능을 포함하지 않는다.

## 상품 데이터 기준

각 상품은 최소한 다음 정보를 가진다.

- 내부 식별자
- 상품명
- 분류: 반지, 목걸이, 예물, 순금 등
- 대표 및 추가 사진
- 짧은 소개
- 원화 가격
- 가격 확인 시점
- 소재, 중량, 옵션 및 추가 비용 설명
- 상태: 초안, 공개, 품절, 숨김
- 대표 상품 여부와 노출 순서
- 생성 및 최종 수정 시점

공개 상태로 바꿀 때 상품명, 분류, 대표 사진, 가격, 가격 확인 시점이 없으면 저장을 막는다. 가격은 계산 오류를 피할 수 있도록 문자열이 아닌 0 이상의 정수 원 단위로 저장한다.

가격 확인 시점부터 14일이 지나면 원본 콘텐츠를 삭제하지 않고 공개 사이트의 상품 목록과 검색 결과에서 제외한다. 관리 화면에서는 `가격 확인 필요` 항목을 쉽게 찾을 수 있어야 하며, 가격과 확인일을 갱신하면 다음 배포에서 다시 노출한다.

정적 사이트에서도 날짜 경과에 맞춰 숨김이 적용되도록 하루 1회 예약 배포를 실행한다. 예약 작업은 공개된 배포 훅 주소를 코드에 넣지 않고 저장소 비밀값으로 관리한다. 콘텐츠 게시 시 실행되는 배포와 예약 배포 모두 같은 검증 및 생성 절차를 사용한다.

## 점포 정보 데이터 기준

점포 정보는 상품과 별도로 한 곳에서 관리한다.

- 점포명
- 주소
- 전화번호
- 요일별 영업시간
- 정기 휴무일
- 임시 휴무 또는 영업시간 변경 공지
- 카카오톡 채널 링크와 활성화 여부
- 지도 링크

카카오톡 버튼은 활성화 상태이고 유효한 링크가 있을 때만 표시한다. 영업시간의 기본값은 월요일부터 토요일 10:00~18:00이며, 매주 일요일은 휴무다.

## 관리자와 보안

- 관리자 계정은 1개를 기본으로 한다.
- 공개 사이트와 관리 화면의 권한을 분리한다.
- 관리 계정에는 강한 비밀번호와 가능한 경우 다중 인증을 사용한다.
- 관리 비밀값을 브라우저 코드나 저장소에 포함하지 않는다.
- 콘텐츠 변경 이력 또는 복구 가능한 백업을 유지한다.

## 초기 권장 방향

- 온라인 결제 없이 빠르게 공개할 수 있는 콘텐츠 관리형 웹사이트
- 방문자에게는 빠른 반응형 랜딩 페이지, 점주에게는 별도의 관리 화면 제공
- 이미지 최적화와 기본 검색 엔진 메타데이터 제공
- 상품 데이터와 화면 코드를 분리
- 관리 시스템 장애가 발생해도 이미 공개된 사이트는 가능한 한 계속 표시
- 문의는 별도 서버 개발보다 전화를 우선 연결하고, 카카오톡 채널 개설 후 해당 링크를 추가

초기 운영비는 무료 플랜 범위 내에서 시작한다. 무료 한도와 서비스 정책은 변경될 수 있으므로 공개 전과 정기 점검 시 공식 가격 및 한도 문서를 다시 확인한다.
