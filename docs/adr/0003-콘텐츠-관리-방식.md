# 콘텐츠 관리 방식

- 상태: 확정
- 날짜: 2026-06-18

## 배경

상품 사진과 소개 내용을 매주 변경할 예정이다. 매번 개발자가 파일과 코드를 수정하는 방식은 업데이트를 늦추고 지속적인 관리 비용을 만든다.

## 결정

상품과 영업 정보는 화면 코드와 분리해 콘텐츠 관리 시스템에서 관리한다. 관리자는 코드를 수정하지 않고 상품을 추가, 수정, 숨김 처리하고 노출 순서를 바꿀 수 있어야 한다.

## 이유

- 점주가 원하는 시점에 상품을 갱신할 수 있다.
- 반복적인 개발 작업을 줄일 수 있다.
- 상품 정보의 구조와 표시 방식을 일관되게 유지할 수 있다.

## 부담과 후속 작업

- 관리 계정과 권한을 안전하게 운영해야 한다.
- 사진 업로드 및 게시 방법을 익혀야 한다.
- 콘텐츠 관리 제품의 비용과 장기 운영 가능성을 검토해야 한다.
- 수정에 주로 사용할 기기와 관리자 수를 확인한 뒤 제품을 선택한다.
