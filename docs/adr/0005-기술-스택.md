# 기술 스택

- 상태: 확정
- 날짜: 2026-06-18

## 배경

남원금방 사이트는 한 명의 관리자가 컴퓨터에서 매주 상품과 가격을 갱신해야 한다. 온라인 주문과 결제는 필요하지 않으며 초기 운영비는 무료를 우선한다. 방문자 페이지는 모바일에서 빠르고 검색 엔진이 이해하기 쉬워야 한다.

## 결정

- Astro와 TypeScript로 방문자 사이트를 개발한다.
- Sanity Content Lake에 상품과 점포 정보를 저장한다.
- Sanity Studio를 관리 화면으로 사용한다.
- Cloudflare Pages에 정적 사이트를 배포한다.
- GitHub에 소스 코드를 보관하고 배포를 연결한다.
- Sanity 게시 시 웹훅으로 Cloudflare Pages 재배포를 요청한다.
- 하루 1회 예약 배포를 실행해 14일이 지난 가격의 상품을 공개 결과에서 제외한다.

## 선택 이유

- Astro는 콘텐츠 중심 정적 사이트를 생성하기 적합하다.
- Sanity Studio는 별도 관리 화면을 처음부터 개발하지 않고 구조화된 상품 편집 기능을 제공한다.
- Sanity 무료 플랜은 관리자 1명과 공개 상품 콘텐츠에 필요한 초기 범위를 제공한다.
- Cloudflare Pages 무료 플랜의 빌드 한도는 매주 콘텐츠를 갱신하는 운영 방식에 충분하다.
- 정적 배포는 관리 시스템에 일시적인 문제가 생겨도 마지막으로 성공한 사이트를 계속 제공할 수 있다.

## 무료 플랜에서 주의할 점

- Sanity 무료 플랜의 데이터셋은 공개형이므로 개인정보와 비밀값을 저장하지 않는다.
- 무료 한도와 정책은 바뀔 수 있으므로 정기적으로 확인한다.
- 사용자 도메인을 구매하면 도메인 등록 비용은 별도로 발생한다.
- 무료 한도를 넘기기 전에 사용량을 확인하고 유료 전환 여부를 결정한다.

## 검토한 대안

- 코드와 정적 파일만 사용: 비용은 낮지만 매주 상품을 수정하기 어렵다.
- 전용 관리 화면 직접 개발: 원하는 기능을 만들 수 있지만 초기 개발과 보안 유지 비용이 크다.
- 쇼핑몰 서비스: 결제와 재고 관리에는 유리하지만 현재 범위에는 기능과 비용이 과하다.
